① Login ke Dashboard

Buka URL dashboard di browser. Anda akan melihat halaman login.

1
Masukkan Username (tanpa @narasi.tv, cukup nama depan) dan Password yang diberikan admin.
2
Klik Masuk ke Dashboard atau tekan Enter.
3
Dashboard akan terbuka langsung dengan data lokal/cache. Data terbaru dari server dimuat di background dan akan otomatis menggantikan tampilan begitu selesai — tidak perlu menunggu atau refresh manual.
💡 Lupa password? Hubungi admin untuk reset password di Supabase Dashboard → Authentication → Users. Login menggunakan Supabase Auth dengan format email username@narasi.tv di balik layar — Anda hanya perlu mengetik username-nya saja di form login.
🔒 Sesi otomatis tersimpan. Setelah login sekali, sesi Anda tetap aktif selama tab browser tidak ditutup/sesi tidak dihapus. Tidak perlu login ulang setiap kali membuka dashboard di tab yang sama.
② Halaman Dashboard Utama

Dashboard utama dirancang dengan dua pasangan filter → konten yang independen satu sama lain — penting untuk dipahami supaya tidak salah duga saat mengubah filter:

🗓 Periode Aktif → Summary Cards

Di bagian paling atas, ada dropdown "Periode Aktif" yang bisa dipilih bebas (minggu/bulan apa saja yang sudah lewat atau sedang berjalan). Dropdown ini mengontrol langsung angka di Summary Cards di bawahnya — pilih periode lain, dan kartu ringkasan akan menghitung ulang berdasarkan data periode tersebut.

💡 Periode yang sedang berjalan ditandai dengan label "(berjalan)" di dropdown.
📈 Filter Rentang Grafik → Grafik Tren

Terpisah dari dropdown di atas, ada panel "Rentang Grafik Tren" yang menyatu langsung dengan grafiknya (satu kotak, dipisahkan garis tipis) — supaya jelas bahwa filter ini cuma mempengaruhi grafik di bawahnya, bukan Summary Cards. Opsinya:

•
Weekly / Monthly — tipe periode yang ditampilkan di grafik.
•
4 / 8 / 12 Periode Terakhir — preset cepat.
•
Rentang Kustom — pilih periode awal dan akhir secara bebas lewat dua dropdown "Dari" dan "Sampai".

Grafik menampilkan satu tab per Metrics Group yang sudah dikonfigurasi (lihat bagian ⑨ Konfigurasi Dashboard).

📋 KPI per Anggota Tim

Di bagian paling bawah halaman Dashboard, ada kartu input KPI untuk setiap anggota tim — ini bagian yang dipakai untuk mengisi data harian/mingguan (dijelaskan detail di bagian berikutnya).

③ Input KPI per Anggota Tim

Di bagian bawah Dashboard terdapat kartu KPI untuk setiap anggota tim. Setiap anggota mengisi KPI-nya sendiri, dan periode toggle Weekly/Monthly di sini independen dari filter manapun di bagian atas halaman.

Cara Input KPI Mingguan
1
Pastikan tab 📅 Weekly aktif di kartu anggota yang ingin diisi.
2
Dropdown periode otomatis menunjuk ke minggu berjalan (Senin–Minggu). Untuk mengisi minggu lain, pilih dari dropdown.
3
Jika belum ada data, klik ✏️ Isi KPI. Form input akan muncul.
4
Isi kolom Aktual dengan nilai pencapaian nyata. Kolom Target sudah terisi default dari Pengaturan KPI — ubah jika perlu kasus khusus.
5
Kolom Status akan otomatis berubah: ● On Track, ● Warning, atau ● Off Track.
6
Kolom vs Lalu menampilkan perbandingan dengan periode sebelumnya secara otomatis (butuh data periode sebelumnya).
7
Klik 📨 Simpan Data untuk menyimpan ke histori dan Supabase.
Cara Input KPI Bulanan

Sama seperti mingguan — klik tab 📆 Monthly di kartu anggota, lalu ikuti langkah yang sama.

Edit Data yang Sudah Disubmit

Pilih periode yang sudah punya data dari dropdown → data tampil dalam mode read-only → klik ✏️ Edit Data di pojok kanan bawah tabel → ubah nilai → klik ✓ Perbarui Data.

💡 Jika Anda submit ulang periode yang sudah ada data, akan muncul konfirmasi "Perbarui Data?". Data lama akan ditimpa (bukan duplikat) — sistem menghapus entri lama untuk periode tersebut lalu menyimpan yang baru.
Kolom Summary di Bawah Tabel

Setelah tabel KPI, muncul ringkasan otomatis yang menunjukkan tiap metrik: apakah melampaui target, di bawah target, dan naik/turun berapa persen dibanding periode sebelumnya.

⚡ Sinkronisasi multi-perangkat: Setiap submit langsung tersimpan ke Supabase. Browser lain yang sedang terbuka akan menerima data terbaru secara otomatis dalam ±30 detik (polling background), tanpa perlu refresh manual dan tanpa risiko saling menimpa data antar anggota.
④ Histori KPI

Tab 🕓 Histori KPI menampilkan rekap semua data yang sudah disubmit, lintas semua anggota dan periode.

•
Filter — saring berdasarkan periode atau anggota tim menggunakan dropdown filter di atas tabel.
•
Edit — klik ✏️ pada baris untuk membuka modal edit dan mengubah nilai aktual, target, atau catatan.
•
Hapus — klik ✕ untuk menghapus entri dari histori. Data di Supabase juga ikut terhapus secara permanen.
•
Export CSV — klik tombol ⬇ Export CSV di toolbar tab Histori (kanan atas tabel) untuk mengunduh data histori yang sedang ditampilkan, sesuai filter aktif.
🔄 Ikon "Target Mengikuti Terkini"

Jika Anda melihat ikon 🔄 di sebelah nilai Target pada sebuah baris, itu berarti target untuk periode tersebut tidak pernah diisi secara eksplisit saat data disubmit dulu — sehingga baris itu otomatis mengikuti target global yang berlaku saat ini di Pengaturan KPI. Begitu target global direvisi lagi, nilai pada baris ini akan ikut berubah secara otomatis tanpa perlu tindakan tambahan.

Baris tanpa ikon 🔄 berarti target-nya sudah dikunci (freeze) — baik karena diisi eksplisit saat submit, atau karena sudah pernah diterapkan lewat fitur Revisi Target Retroaktif (lihat bagian ⑦).

⚠️ Data histori dimuat ulang dari Supabase setiap kali tab Histori dibuka. Jika data terasa belum lengkap, tunggu sebentar — proses sinkronisasi background berjalan otomatis.
⑤ Proyeksi & Forecasting

Tab 🔮 Proyeksi & Forecasting memungkinkan prediksi nilai KPI ke depan berdasarkan data historis nyata yang sudah disubmit dari Dashboard.

ℹ️ Syarat minimal data: sebuah metrik hanya akan muncul di dropdown forecast jika sudah punya minimal 2 periode data yang disubmit (Weekly atau Monthly, dihitung terpisah). Dropdown akan menampilkan keterangan jumlah periode yang tersedia untuk setiap metrik, contoh: "Organic Traffic (Weekly) — 4 periode data". Metrik yang datanya belum cukup tidak akan muncul di pilihan.
Cara Menggunakan
1
Pilih metrik yang ingin di-forecast dari dropdown.
2
Pilih metode forecasting:
📈 Linear Regression — tren garis lurus. Cocok untuk data yang naik/turun konsisten.
〰️ Moving Average — rata-rata 3 periode terakhir. Cocok untuk data stabil.
🔄 Exp. Smoothing (Holt) — bobot lebih besar ke data terbaru. Cocok untuk data dengan tren yang berubah.
🚀 Growth Rate (CAGR) — proyeksi berbasis rata-rata pertumbuhan historis. Cocok untuk data yang tumbuh konsisten.
3
Pilih horizon — berapa periode ke depan yang ingin diproyeksikan: pilihan cepat 3, 6, 9, atau 12 periode, atau pilih Kustom... untuk memasukkan angka bebas antara 1 sampai 52 periode ke depan lewat kolom input yang muncul.
4
Klik 🔮 Jalankan Forecast.
5
Lihat grafik, tabel detail dengan confidence level per periode, dan insight otomatis. Label periode proyeksi berupa tanggal kalender asli (misal "6-12 Jul 2026"), bukan label generik — jadi langsung bisa dibaca konteksnya.
6
Klik ⬇ Export Hasil untuk mengunduh hasil forecast sebagai CSV — file akan menyesuaikan otomatis dengan horizon yang dipilih, termasuk horizon kustom yang panjang.
💡 Confidence level menurun seiring bertambahnya horizon. Proyeksi periode-periode awal jauh lebih andal daripada proyeksi 30-50 periode ke depan — gunakan horizon panjang sebagai gambaran kasar tren jangka panjang, bukan angka pasti.
⑥ Pengaturan KPI per Anggota

Tab ⚙️ Pengaturan KPI → KPI per Anggota untuk mengatur metrik, target, dan tipe data tiap anggota.

•
Pilih anggota — klik tab nama anggota di bagian atas.
•
Nama Metrik — ubah nama metrik sesuai kebutuhan.
•
Tipe Data — pilih 💰 Rupiah, 📊 Persen (%), atau 🔢 Angka. Tipe ini menentukan format tampilan dan placeholder input.
•
Target Weekly / Monthly — isi nilai target untuk tiap periode. Mengubah nilai ini akan otomatis berlaku untuk semua periode masa depan yang belum disubmit.
•
↓ Lebih Kecil = Baik — centang untuk metrik seperti CPC atau Ad Spend. Status on track akan dihitung terbalik.
•
Centang aktif (✓) — hapus centang untuk menonaktifkan metrik tanpa menghapusnya.
•
+ Tambah Metrik — tambah baris metrik baru untuk anggota tersebut.
•
✕ — hapus metrik (tidak bisa di-undo).
⚠️ Klik ✓ Simpan Pengaturan di pojok kanan atas setelah selesai mengubah. Perubahan belum tersimpan sebelum diklik.
🔄 Soal periode yang sudah lewat: mengubah target di sini tidak otomatis mengubah histori yang sudah disubmit dengan target eksplisit (lihat penjelasan ikon 🔄 di bagian ④ Histori). Jika Anda ingin merevisi target untuk periode yang sudah lewat, gunakan fitur Revisi Target Retroaktif — lihat bagian ⑦.
⑦ Revisi Target Retroaktif

Fitur ini menjawab skenario: "Target sudah saya isi sejak awal, tapi sekarang saya sadar nilainya kurang tepat — saya ingin merevisinya, termasuk untuk periode-periode lama yang sudah disubmit."

Kapan Dialog Ini Muncul

Setiap kali Anda mengubah nilai target sebuah metrik di Pengaturan KPI dan klik ✓ Simpan Pengaturan, sistem otomatis memeriksa: apakah ada histori lama untuk metrik tersebut yang target-nya sudah eksplisit terisi (bukan kosong) dan nilainya benar-benar berbeda dari yang baru? Jika tidak ada perubahan relevan, proses simpan berjalan seperti biasa tanpa dialog tambahan apa pun.

Jika ada, sebuah dialog "Terapkan Revisi Target ke Histori" akan muncul setelah pengaturan tersimpan.

Cara Menggunakan Dialog
1
Dialog menampilkan daftar periode yang terdampak, dikelompokkan per periode, lengkap dengan metrik dan PIC yang terpengaruh serta perbandingan target lama → target baru.
2
Centang baris yang ingin diikutkan revisi — bisa per metrik satu-satu, atau centang checkbox di header periode untuk menerapkan ke semua metrik dalam periode itu sekaligus.
3
Gunakan tombol ✓ Pilih Semua / ✕ Kosongkan Semua untuk seleksi cepat lintas periode dan metrik.
4
Klik Terapkan ke Periode Terpilih — hanya baris yang dicentang yang nilainya berubah; sisanya tetap memakai target lama (freeze).
5
Atau klik Lewati jika ternyata tidak ada yang perlu direvisi — semua histori tetap dengan target lama.
💡 Tidak ada efek permanen sebelum Anda klik Terapkan — aman untuk membuka dialog ini, melihat-lihat daftarnya, lalu menutup tanpa mengubah apa pun.
ℹ️ Fitur ini khusus untuk target yang sudah pernah diisi sebelumnya. Jika target dulunya kosong (belum pernah diset), histori tersebut sudah otomatis mengikuti target terkini tanpa perlu dialog ini — lihat penjelasan ikon 🔄 di bagian ④ Histori.

⑪ Sinkronisasi & Arsitektur Supabase

Dashboard ini sepenuhnya berjalan sebagai file HTML statis (bisa dihosting di GitHub Pages, Netlify, atau platform sejenis) dan berkomunikasi langsung ke Supabase — sebuah backend-as-a-service berbasis PostgreSQL — lewat REST API dan Supabase Auth. Tidak ada Google Sheets atau Apps Script yang terlibat lagi.

Login & Autentikasi

Login menggunakan Supabase Auth (email + password). Username yang diketik di form login otomatis ditambahkan suffix @narasi.tv sebelum dikirim ke server — jadi anggota tim hanya perlu mengetik username pendeknya saja.

Sinkronisasi Real-Time (Polling)

Setelah login, dashboard menjalankan pengecekan data terbaru ke Supabase setiap 30 detik di background. Ini memastikan:

•
Data yang disubmit anggota lain muncul otomatis tanpa perlu refresh manual.
•
Submit dari dua browser/perangkat berbeda untuk PIC atau periode yang berbeda tidak saling menimpa — setiap submit hanya mempengaruhi data PIC dan periode miliknya sendiri.
•
Jika dua orang submit untuk PIC dan periode yang sama persis dalam rentang waktu yang berdekatan, submit yang lebih baru akan menjadi nilai final (last-write-wins) — koordinasikan dengan tim jika ini berpotensi terjadi.
💡 Data dimuat dari Supabase di background setelah login dan setiap kali tab Histori/Dashboard dibuka. Proses ini tidak memblokir tampilan — dashboard langsung tampil dengan data lokal, lalu otomatis ter-update begitu data terbaru selesai dimuat.
⑫ FAQ — Pertanyaan Umum
Data yang saya input hilang setelah refresh halaman.
Pastikan Anda sudah klik 📨 Simpan Data sebelum menutup atau refresh halaman. Data yang belum disubmit hanya tersimpan sementara di memori browser dan akan hilang saat halaman dimuat ulang.
Kolom "vs Lalu" kosong / menampilkan "–".
Kolom ini membutuhkan data dari periode sebelumnya. Pastikan periode sebelumnya sudah diisi dan disubmit terlebih dahulu.
Summary Card dan grafik tidak muncul / menampilkan "Belum ada Metrics Group".
Buka ⚙️ Pengaturan KPI → 📊 Konfigurasi Dashboard, buat Metrics Group, tambahkan metrik, lalu klik Simpan Konfigurasi.
Muncul error "Tidak bisa terhubung ke server" atau "Invalid login credentials" saat login.
Periksa koneksi internet. Jika koneksi normal, kemungkinan kredensial Supabase di kode (Project URL / API Key) belum diisi dengan benar, atau username yang diketik tidak cocok dengan akun yang dibuat di Supabase Authentication. Hubungi admin/tim teknis jika berlanjut.
Forecast menampilkan "Belum cukup data historis" padahal sudah ada data.
Forecast butuh minimal 2 periode data yang sudah disubmit untuk metrik dan tipe periode (Weekly/Monthly) yang sama. Submit data untuk minimal satu periode tambahan, lalu coba lagi — dropdown metrik akan otomatis menampilkan opsi begitu syarat ini terpenuhi.
Periode minggu/bulan berjalan tidak muncul di dropdown.
Dashboard otomatis memperbarui daftar periode berdasarkan tanggal hari ini setiap kali dibuka. Coba refresh halaman untuk memastikan daftar periode up-to-date, terutama jika tab dibuka sejak beberapa hari lalu dan belum ditutup.
Saya submit KPI dua kali untuk periode yang sama.
Dashboard akan menampilkan konfirmasi "Perbarui Data?" — data lama akan ditimpa dengan data baru tanpa duplikasi di Supabase, karena sistem menghapus entri lama untuk PIC+periode+metrik tersebut sebelum menyimpan yang baru.
Target di histori lama tiba-tiba berubah padahal saya tidak mengeditnya.
Ini perilaku normal jika baris tersebut bertanda ikon 🔄 — artinya target periode itu belum pernah diisi eksplisit saat submit dulu, sehingga otomatis mengikuti target global terkini. Jika target global di Pengaturan KPI direvisi, baris ini ikut menyesuaikan. Lihat bagian ④ Histori untuk detail.
Anggota baru tidak bisa login walau sudah ditambahkan di Anggota Tim.
Menambahkan anggota di tab Anggota Tim hanya membuat profilnya tampil di Dashboard — belum membuat akun login. Admin perlu membuat akun secara terpisah di Supabase Dashboard → Authentication → Users dengan email username@narasi.tv, lalu (opsional) menambahkan nama tampilannya ke tabel kpi_users.
Bagaimana cara logout?
Klik tombol Keluar di pojok kanan atas topbar. Sesi akan berakhir dan Anda kembali ke halaman login.
Project Supabase tidak merespon / data tidak ter-load sama sekali.
Project Supabase tier gratis akan otomatis "pause" setelah 7 hari tanpa aktivitas. Buka Supabase Dashboard untuk mengaktifkannya kembali — proses ini biasanya hanya butuh beberapa menit. Aktivitas rutin tim (submit data mingguan) biasanya sudah cukup mencegah hal ini terjadi.
